## Entry 4: Hardware Testing Strategy and Breadboard Planning

**Date:** 4/3/26  

**Objective:** Plan the hardware tests needed before full PCB integration and decide how to use breadboard testing effectively.

**Work Completed:**  
The team discussed PCB printing, ESP32 programming, power wiring, and whether to begin testing on a breadboard while waiting for PCB hardware. We identified that breadboard testing could help verify the most important functions early, including pump direction, pump flow rate, motor driver logic, load-cell communication, and ESP32 pin behavior. We also discussed the importance of matching breadboard pins to expected PCB pins as much as possible so that test code could later transfer to the final board with minimal changes.

**Results / Decisions:**  
The team decided that breadboard testing was necessary to reduce schedule risk. Waiting for the PCB before testing every subsystem would create too much uncertainty. The plan was to build modular tests first, then later merge working subsystem code into the final integrated homebase firmware.

**Next Step:**  
Begin breadboard tests for pump drivers and load cells, then record results for verification and debugging.
