## Entry 2: PCB Order and Hardware Availability Planning

**Date:** 3/25/26  

**Objective:** Track PCB ordering deadlines and plan around expected hardware availability.

**Work Completed:**  
The team reviewed the PCB ordering timeline and the fourth-round JLCPCB order deadline. Since the final system depended on custom PCB hardware, we discussed which circuits needed to be finalized and what could still be tested on a breadboard. The hardware plan included preparing for ESP32 bring-up, pump driver testing, load-cell amplifier testing, and UV/photodiode circuit validation. We also discussed the importance of matching breadboard pinouts to the PCB schematic so that code written during testing could transfer more easily to the final PCB.

**Results / Decisions:**  
The team decided that code should be written with PCB pin assignments in mind whenever possible, even if the immediate test was performed on a breadboard. This would reduce later rework when switching from breadboard hardware to the PCB. The team also recognized that some hardware testing would need to happen before the final PCB arrived to keep the schedule realistic.

**Next Step:**  
Confirm the latest schematic pin assignments and begin writing test code that can be reused for PCB bring-up.
