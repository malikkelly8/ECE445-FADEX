## Entry 5: Mobile Application and Adaptive Algorithm Design

**Date:** 2/27/26  

**Objective:** Define the role of the mobile application and describe how user behavior would affect the tapering algorithm.

**Work Completed:**  
The team wrote the mobile application subsystem section. The app was planned as the main user interface for taper schedule setup, system monitoring, and usage history. It would communicate with the Home Base and Handheld through BLE, receive puff duration and frequency data, display device status, and send target concentration information to the Home Base. The design document also described an adaptive tapering requirement where normal puff density would produce a larger concentration reduction and high puff density would slow the taper to avoid withdrawal or relapse.

**Results / Decisions:**  
The team decided that the app should provide user-facing controls and analytics, but the embedded firmware should execute the actual dosing and safety decisions. This separation was important because app connectivity might be interrupted, while pump and coil safety needed to remain reliable. The adaptive algorithm became a major software verification item, requiring simulation tests to confirm concentration changes under normal and high puff-density conditions.

**Next Step:**  
Develop a software prototype or simulation that can test puff-density logic and verify that concentration updates match the expected taper behavior.
