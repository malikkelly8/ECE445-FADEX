## Entry 3: Algorithm Flowchart and Documentation Update

**Date:** 3/27/26  
**Objective:** Improve documentation of the adaptive tapering algorithm and make the algorithm easier to explain for future reports and presentations.

**Work Completed:**  
The team worked on making the algorithm easier to visualize and explain. The algorithm was organized around several major steps: detecting puff activity, measuring puff duration and count, collecting user state information, computing the next concentration, applying safety limits, calculating pump volumes, and updating the user interface. This helped connect the software algorithm to the physical hardware subsystems, since puff data came from the handheld, concentration calculations ran on the homebase, and final dosing was performed by pumps and verified with sensors.

**Results / Decisions:**  
The team decided that the algorithm should be presented as a staged flow rather than just equations. This made it easier to explain how hardware inputs influenced the final concentration. The flowchart also helped reveal which variables needed to be displayed in the app or serial interface, including current concentration, next concentration, puff count, puff duration, scenario code, and dose volumes.

**Next Step:**  
Create or refine an algorithm flowchart and use it to guide software implementation.
