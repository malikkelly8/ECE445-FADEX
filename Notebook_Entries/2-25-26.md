## Entry 3: Control, Safety Logic, and Connectivity Design

**Date:** 2/25/26  

**Objective:** Document how the ESP32 control system would coordinate dosing, safety logic, wireless communication, and user data.

**Work Completed:**  
The team wrote the control, safety logic, and connectivity subsystem section. The Home Base ESP32 was assigned responsibility for calculating and executing precision fluid mixing, while the Handheld ESP32 was responsible for responsive puff detection and safe coil activation. The design document specified that tapering algorithms should be stored in nonvolatile memory so that the user’s schedule would not be lost during a power cycle. The team also documented BLE communication goals, sensor inputs, pump outputs, and safety functions such as coil cutoff timing and temperature-based shutdown.

**Results / Decisions:**  
The team decided that safety-critical logic should remain in firmware rather than being controlled only by the mobile app. This included pump control, coil firing limits, and lockout behavior. The design also required the ESP32 logic rail to remain stable during wireless communication and actuator activity, which linked the control subsystem directly to power subsystem verification.

**Next Step:**  
Build test plans for ESP32 logic stability, coil cutoff timing, BLE latency, and algorithm response to simulated puff behavior.
