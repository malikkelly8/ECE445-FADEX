## Entry 2: Project Direction and System Planning

**Date:** 2/4/26  

**Objective:** Convert the early FadeX idea into a clearer project plan that could be explained for project approval.

**Work Completed:**  
After the third class meeting, the team refined the problem statement and discussed how FadeX would address nicotine tapering in a safer and more controlled way. We focused on the idea that a fixed taper schedule may not respond well to real user behavior, so the device should collect usage data and adjust the nicotine concentration over time. The team also started outlining the main engineering problems: detecting puff activity, calculating a target concentration, physically mixing the correct ratio of concentrate and diluent, and providing a user-facing interface.

**Results / Decisions:**  
The project direction became more focused around an adaptive homebase system rather than only a modified vape device. The team decided that objective usage data and user input should both influence the tapering algorithm. This became an important design decision because it tied the software algorithm directly to the hardware requirements for sensing, pumping, and communication.

**Next Step:**  
Prepare the project approval submission and begin listing required components, including ESP32 boards, pumps, sensors, and possible concentration measurement methods.
