## Entry 4: Blynk Dashboard and Chatbot Demo Flow

**Date:** 4/23/26  

**Objective:** Improve the user-facing demo by using Blynk and the FadeX Chatbot to guide the dose workflow.

**Work Completed:**  
I worked on the app/demo flow using Blynk virtual pins and the FadeX Chatbot terminal interface. The dashboard included values such as current concentration, next concentration, puff count, puff duration, survey score, trigger code, scenario code, nicotine volume, diluent volume, money saved, milestone progress, clinician override, prime command, run dose command, and puff button. The chatbot was designed to guide the user through a session by showing current status, collecting survey input, asking about triggers only when needed, and preparing the dose calculation. I also worked through issues with terminal clearing, repeated messages, and the order of chatbot prompts.

**Results / Decisions:**  
The team decided that the Blynk dashboard would be the main app interface, while the chatbot would make the demo easier to follow. The app would display system values, and the chatbot would explain what the system was doing step by step. However, the homebase ESP32 would still remain responsible for dosing logic and safety-critical pump control.

**Next Step:**  
Refine the chatbot prompt order, reduce repeated terminal output, and ensure the run-dose workflow asks the right questions before calculating the next concentration.
