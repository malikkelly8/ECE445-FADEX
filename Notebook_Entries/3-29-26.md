## Entry 4: Breadboard-to-PCB Transition Planning

**Date:** 3/29/26  

**Objective:** Plan how breadboard testing would transition into PCB testing without changing the software structure too much.

**Work Completed:**  
The team discussed how to organize code and hardware tests so that the same firmware concepts could work on both breadboard and PCB setups. We identified that pin definitions should be isolated near the top of each sketch so that changing from breadboard pins to PCB pins would be straightforward. We also discussed the need for small subsystem sketches, such as pump tests, load-cell calibration tests, UV LED PWM tests, photodiode ADC tests, and ESP32 communication tests.

**Results / Decisions:**  
The team decided that subsystem test sketches would be important for debugging. Instead of immediately building one large final program, the project would benefit from smaller test programs that each verified a single hardware function. This decision later made it easier to debug pumps, load cells, ESP-NOW, Blynk, and UNCS separately before integration.

**Next Step:**  
Begin creating skeleton code tests for each hardware subsystem and prepare for the individual progress report due the following week.
