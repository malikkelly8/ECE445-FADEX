## Entry 1: Design Document Structure and Updated System Architecture

**Date:** 2/23/26 

**Objective:** Convert the project proposal into a more complete design document with a clearer system architecture, subsystem breakdown, and verification plan.

**Work Completed:**  
The team worked on expanding the proposal into the formal design document. The design was updated from a general dual-reservoir vape concept into a two-part architecture consisting of a stationary Home Base and a passive Handheld Vaporizer. The Home Base was responsible for mixing nicotine and diluent into a central chamber, while the Handheld Vaporizer was intended to preserve the familiar user experience of a vape while receiving a prepared daily dose. The design document included the problem statement, proposed solution, visual aids, high-level requirements, block diagram, subsystem descriptions, tolerance analysis, cost analysis, schedule, and safety/ethics discussion.

**Results / Decisions:**  
The team decided that separating the complex mixing hardware from the handheld device would make the system more feasible, safer, and easier to test. The Home Base would handle pumps, reservoirs, load cells, and concentration verification, while the Handheld would mainly handle user puff detection, vapor generation, and usage logging. This decision reduced the size and complexity of the handheld while allowing the Home Base to perform more accurate fluid control.

**Next Step:**  
Finish detailed subsystem requirements and make sure each major requirement has a corresponding verification method.
