## Entry 2: Algorithm Direction and User Data Planning

**Date:** 3/18/26  

**Objective:** Refine how user behavior data would influence the FadeX tapering algorithm.

**Work Completed:**  
The team continued thinking through how the algorithm should respond to user behavior. The design document already included the idea that puff density and usage patterns could affect the tapering rate, but the exact structure still needed to be refined. We discussed using puff count, puff duration, and possibly self-report feedback to decide whether the system should continue tapering, slow the taper, or temporarily stabilize the concentration. This helped connect the software design to the real behavioral problem that FadeX is trying to solve.

**Results / Decisions:**  
The team decided that the algorithm should not only follow a preset schedule. It should also be able to respond to signs of heavy use or relapse risk. This idea later supported the use of scenario codes and survey-style feedback in the app/chatbot interface. The main decision was that the tapering system should remain adaptive rather than purely linear.

**Next Step:**  
Develop a clearer algorithm flowchart and begin defining the variables needed for concentration calculations.
