## Entry 2: Mock Presentation Slide Planning

**Date:** 4/21/26  

**Objective:** Build a short mock presentation that explained the project objective, algorithm, and hardware flow clearly.

**Work Completed:**  
The team divided mock presentation responsibilities across the required slide categories: title, introduction, objective, design, and conclusion. I focused on the objective slide and the algorithm-based design slides. The presentation flow was planned around explaining the high-level algorithm first, then breaking down each hardware subsystem based on where it appears in the algorithm. This included puff detection and usage tracking, adaptive decision logic, fluid delivery with pumps/load cells, and concentration feedback using the UV/photodiode system.

**Results / Decisions:**  
The team decided that the presentation should explain the algorithm as the main story and then show how the hardware supports each step. This made the presentation more coherent than listing hardware blocks separately. It also helped connect the user problem, software algorithm, and physical mixing system in a way that non-specialists could follow.
