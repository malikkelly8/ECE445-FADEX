## Entry 3: Blynk App and User Interface Planning

**Date:** 4/15/26  

**Objective:** Develop the app-side plan for displaying system status and collecting user inputs during the demo.

**Work Completed:**  
I worked on planning the Blynk dashboard and how it would support the FadeX demo. The dashboard needed to show values such as current concentration, next concentration, daily puff count, daily puff duration, survey score, trigger code, scenario code, nicotine volume, diluent volume, milestone progress, and money saved. I also considered how the app could be used to start a dose run, prime pumps, and display algorithm outputs. Since app connectivity could become a risk, I also considered using the serial monitor as a backup control interface.

**Results / Decisions:**  
The team decided that the app should be useful for user-facing interaction but that the ESP32 firmware should remain responsible for safety-critical dosing. This meant the app could collect inputs and display outputs, while the homebase controller would still calculate and enforce dose behavior. Blynk was selected as the main app option because it allowed quick dashboard creation and virtual pins for data exchange.

**Next Step:**  
Create Blynk datastreams, connect the ESP32 to the dashboard, and test whether dashboard values update reliably.
