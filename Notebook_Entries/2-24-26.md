## Entry 2: High-Level Requirements and System Verification Planning

**Date:** 2/24/26  

**Objective:** Define measurable high-level requirements for the full FadeX system and connect them to verifiable tests.

**Work Completed:**  
The team refined the high-level requirements for the design document. The first requirement focused on asynchronous volumetric mixing precision, requiring the dual-pump fluidic subsystem to achieve a commanded nicotine-to-diluent ratio with less than 15% steady-state error. The second requirement focused on a dynamic usage-responsive tapering algorithm that could adjust nicotine concentration based on user data and a selected taper schedule. The third requirement focused on wireless data communication, requiring usage analytics and algorithm setpoints to synchronize with the mobile application in near real time.

**Results / Decisions:**  
These high-level requirements helped define what the final system needed to prove: accurate mixing, adaptive control, and reliable communication. The team also began connecting each requirement to subsystem-level verification, such as pump volume tests, algorithm simulations, BLE latency measurements, and data logging checks. This helped shift the project from concept design toward measurable engineering validation.

**Next Step:**  
Write detailed requirements and verification procedures for the power, control, fluid, thermal, and mobile application subsystems.
