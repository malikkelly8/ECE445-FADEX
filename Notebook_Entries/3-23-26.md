## Entry 1: Post-Spring-Break Project Reset

**Date:** 3/23/26  

**Objective:** Re-establish project priorities after spring break and identify the most important tasks for hardware and software progress.

**Work Completed:**  
After spring break, the team reviewed the current project status and compared it against the design document schedule. The main focus was shifting from documentation and planning into physical testing and implementation. We reviewed which subsystems needed immediate attention, including ESP32 programming, pump control, load-cell reading, UV LED/photodiode sensing, and app or serial interface development. The team also discussed how the project might need to use breadboard testing while waiting for PCB fabrication or parts delivery.

**Results / Decisions:**  
The team decided that modular testing was still the safest path forward. Rather than waiting for every part of the final system, each subsystem would be tested independently as soon as enough hardware was available. This approach reduced the risk of discovering integration issues too late in the semester.

**Next Step:**  
Begin preparing breadboard tests and update the project task board with post-break priorities.
