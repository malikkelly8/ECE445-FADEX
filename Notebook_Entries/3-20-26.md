## Entry 3: App Interface and User Interaction Planning

**Date:** 3/20/26  

**Objective:** Plan how the user would interact with FadeX through an app or backup interface.

**Work Completed:**  
The team considered what information the user should see and what inputs the system would need to collect. Possible app values included current concentration, next concentration, puff count, puff duration, treatment week, dose volumes, battery or device status, and progress toward quitting. We also considered whether the app should directly control dosing or only provide inputs and display status while the ESP32 firmware handled safety-critical decisions.

**Results / Decisions:**  
The team decided that the app should act as the user interface and data display, but not as the only safety controller. This meant that the Home Base firmware should still be able to enforce pump limits, dose calculations, and emergency stop behavior even if app communication failed. This decision made the system safer and more robust for demo conditions.

**Next Step:**  
Create a dashboard plan and determine which variables need to be displayed or controlled by the app.
